## Page with Visual Editing

A Nuxt page that works for public users (server-rendered from the Sanity CDN) and for content editors inside the Presentation Tool (draft-aware, stega-encoded, click-to-edit overlays active).

---

## When to use this

You need Presentation Tool support — editors should be able to open the page in the studio and click fields to jump to the right document. No ISR caching needed.

---

## Files

- `nuxt.config.ts` — visual editing module config
- `server/middleware/preview-cache-bypass.ts` — prevent preview responses from being cached
- `pages/[slug].vue` — the page component

---

## Implementation

```ts{}[nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxtjs/sanity'],
  sanity: {
    projectId: process.env.NUXT_SANITY_PROJECT_ID,
    dataset: process.env.NUXT_SANITY_DATASET,
    visualEditing: {
      token: process.env.NUXT_SANITY_VISUAL_EDITING_TOKEN,
      studioUrl: process.env.NUXT_SANITY_VISUAL_EDITING_STUDIO_URL,
    },
  },
})
```

```ts{}[server/middleware/preview-cache-bypass.ts]
import { getCookie, setResponseHeader } from 'h3'

export default defineEventHandler((event) => {
  const previewCookie = getCookie(event, 'sanity-preview-id')
  if (!previewCookie) return
  setResponseHeader(event, 'Cache-Control', 'no-store')
})
```

```vue{}[pages/[slug].vue]
<script setup lang="ts">
const route = useRoute()

const query = groq`*[_type == "page" && slug.current == $slug][0]{
  _id,
  title,
  body,
}`

const { data: page, encodeDataAttribute } = await useSanityQuery<Page>(
  query,
  { slug: route.params.slug },
)
</script>

<template>
  <div v-if="page">
    <h1>{{ page.title }}</h1>
    <!-- body rendered as Portable Text — stega encoding in strings enables overlays -->
    <SanityContent :value="page.body" />
  </div>
</template>
```

---

## Notes

::tip
`useSanityQuery` automatically switches between the public CDN (SSR, public users) and the draft-aware preview proxy (Presentation Tool). No conditional logic needed in your component.
::

::tip
`sanity-preview-id` is the cookie set by the module's built-in `/preview/enable` endpoint when an editor opens the Presentation Tool. The bypass middleware ensures editor responses are never stored in any cache layer.
::

::warning
Non-string fields (numbers, images, booleans) are not stega-encoded. Use `encodeDataAttribute` to add explicit `data-sanity` attributes so overlays appear for those fields — see the [query optimization guide](/advanced/query-optimization#usesanityquery--cdn-performance--full-visual-editing).
::
