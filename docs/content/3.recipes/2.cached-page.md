## Cached Public Page

A Nuxt page with long-lived ISR caching and webhook-based invalidation. Content is served from the edge after the first SSR, and the cache is purged automatically when a document is published in Sanity.

---

## When to use this

You need maximum performance for public users (CDN-cached HTML) and guaranteed freshness within seconds of a publish event. No visual editing required.

---

## Files

- `nuxt.config.ts` — route rules + webhook secret
- `server/api/_sanity/revalidate.post.ts` — webhook handler

---

## Implementation

```ts{}[nuxt.config.ts]
export default defineNuxtConfig({
  routeRules: {
    '/pages/**': { swr: 86400 },
    '/blog/**':  { swr: 86400 },
  },
  runtimeConfig: {
    sanityWebhookSecret: '', // set via NUXT_SANITY_WEBHOOK_SECRET
  },
})
```

```ts{}[server/api/_sanity/revalidate.post.ts]
export default defineEventHandler(async (event) => {
  const body = await readBody(event)
  const config = useRuntimeConfig()

  if (event.headers.get('X-Sanity-Webhook-Secret') !== config.sanityWebhookSecret) {
    throw createError({ statusCode: 401 })
  }

  // Platform-specific edge CDN purge (Vercel, Netlify, etc.) — see caching guide
  await purgeCdnCache(body._id)

  // Clear Nitro's internal server cache
  const storage = useStorage('cache')
  await storage.clear()

  return new Response('Purged successfully!', { status: 202 })
})
```

Configure the Sanity webhook in [sanity.io/manage](https://sanity.io/manage) → API → Webhooks:

| Field | Value |
|---|---|
| **URL** | `https://your-site.com/api/_sanity/revalidate` |
| **Trigger on** | Create, Update, Delete |
| **HTTP method** | POST |
| **HTTP headers** | `X-Sanity-Webhook-Secret: <your-secret>` |
| **Projection** | `{ _id, _type, slug }` |

---

## Notes

::tip
Set `NUXT_SANITY_WEBHOOK_SECRET` in your hosting environment to the same secret value used in the webhook header. Nuxt maps it to `runtimeConfig.sanityWebhookSecret` automatically.
::

::warning
`storage.clear()` removes all Nitro cache entries at once. For large sites with many independently-cached routes, use targeted invalidation — see the [caching guide](/advanced/caching#targeted-nitro-invalidation).
::
