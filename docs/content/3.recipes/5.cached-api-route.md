## Cached API Route

A Nitro API route that fetches from Sanity and caches its JSON response independently of any page. Useful when multiple pages consume the same data (e.g. navigation, product list, global settings).

---

## When to use this

You have a shared data endpoint consumed by several pages via `useFetch`. Caching it at the route level avoids redundant Sanity queries across pages and across server instances.

---

## Files

- `server/api/products.ts` ŌĆö the cached API route
- `server/api/_sanity/revalidate.post.ts` ŌĆö add targeted invalidation (optional)

---

## Implementation

```ts{}[server/api/products.ts]
export default defineCachedEventHandler(async () => {
  const sanity = useSanity()
  return sanity.fetch(
    groq`*[_type == "product"]{ _id, title, "slug": slug.current, price }`,
  )
}, {
  maxAge: 60 * 60 * 24, // 24 hours ŌĆö webhook invalidation handles freshness
  name: 'sanity-products',
})
```

Consume it in a page:

```vue{}[pages/products/index.vue]
<script setup lang="ts">
const { data: products } = await useFetch('/api/products')
</script>
```

### Invalidation

The `storage.clear()` call in the webhook route invalidates `defineCachedEventHandler` entries automatically ŌĆö no changes to the webhook handler are needed.

To remove only this specific entry without clearing the entire cache:

```ts{}[server/api/_sanity/revalidate.post.ts]
const storage = useStorage('cache')
await storage.removeItem('nitro:functions:sanity-products:default')
```

## i18n tip

When the locale comes from a **query parameter** (e.g. `/api/products?lang=en`), Nitro generates a separate cache entry per unique URL ŌĆö no extra configuration needed.

When the locale comes from a **request header** (e.g. `Accept-Language`), Nitro cannot differentiate requests automatically. Use `getKey` to build a cache key that includes the header value:

```ts{}[server/api/products.ts]
export default defineCachedEventHandler(async (event) => {
  const lang = getHeader(event, 'Accept-Language') ?? 'en'
  const sanity = useSanity()
  return sanity.fetch(
    groq`*[_type == "product" && language == $lang]{ _id, title, price }`,
    { lang },
  )
}, {
  maxAge: 60 * 60 * 24,
  name: 'sanity-products',
  getKey: (event) => getHeader(event, 'Accept-Language') ?? 'en',
})
```

## Notes

::tip
`useSanity()` is available in Nitro server routes via the module's auto-imports ŌĆö no manual client setup needed.
::

::tip
`defineCachedEventHandler` entries share the same Nitro `cache` storage used by `routeRules`. The `storage.clear()` in the webhook route therefore clears both page caches and API route caches in a single call.
::
