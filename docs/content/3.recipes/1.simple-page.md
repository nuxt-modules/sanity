## Simple Public Page

A Nuxt page that fetches data from Sanity on the server. No caching, no visual editing. The baseline pattern — useful as a reference when comparing with more advanced recipes.

---

## When to use this

You need a server-rendered page that reads published Sanity content. Start here, then add caching or visual editing as needed.

---

## Files

- `pages/[slug].vue` — the page component

---

## Implementation

```vue{}[pages/[slug].vue]
<script setup lang="ts">
const route = useRoute()

const query = groq`*[_type == "page" && slug.current == $slug][0]{
  _id,
  title,
  body,
}`

const { data: page } = await useSanityQuery<Page>(query, { slug: route.params.slug })
</script>

<template>
  <div v-if="page">
    <h1>{{ page.title }}</h1>
    <SanityContent :value="page.body" />
  </div>
</template>
```

---

## Notes

::tip
On SSR, `useSanityQuery` uses the public Sanity CDN — no auth token, no extra latency, fully CDN-cacheable. Visual editing activates only when an editor opens the page inside the Presentation Tool.
::
